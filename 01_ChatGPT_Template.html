🔎Step 1 — Let’s Find Good Test Events

Before testing “other involvement types”, we need to pick smart sample data.

You already tested:

  • ✅ Arrest (JUV / EXP)
  • ✅ Multiple offenses
  • ✅ Single role

Now we want:

  1. Criminal but NOT arrest (SUS, OFF, VIND, etc.)
  2. Victim-only event
  3. Witness-only event
  4. Multi-role person on same event

Why we can say that with confidence

1. Digital signatures lock form field values at signing time

A PDF digital signature cryptographically seals:

  • All form field values (including dropdowns like STEP)
  • The document byte structure

If a dropdown value (e.g., Step 5 → Step 7) is changed after signing, then one of two things must happen:

  • The signature becomes invalid, or
  • The document is saved as a new file, breaking the original signature chain

There is no “silent” post-signature change possible.

2. Configuration/schema mismatch after update

Very common when:

  • The update changes a database schema
  • The migration partially fails
  • One function uses a newer schema than others

Symptoms:

  • App launches fine
  • Most functions work
  • One action crashes, hangs, or silently fails

📍 How to confirm

  • Compare config files before and after reinstall
  • Look for:
    • Version numbers
    • Feature flags
    • Missing keys
    • JSON/XML schema differences
